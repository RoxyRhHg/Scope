# Scope 底层投资逻辑

> "买股票就是买公司，买公司就是买其未来现金流的折现。"
> — 段永平

Scope 是一个基于段永平投资哲学的 A 股价值筛选与辅助决策系统。本文档定义其底层投资逻辑：从哲学根基到量化落地，从核心框架到具体公式。本文档是系统的"宪法"——所有评分、过滤、排序、推荐逻辑均由此推导。

---

## 1. 投资哲学根基

### 1.1 本分：回到事物本质

**哲学含义**：在任何时候，尤其是有诱惑的时候，排除外界干扰，回到事物本质，辨别是非对错。

**系统落地**：
- 系统的第一优先级永远是"公司本身的质量"，而非"市场在追捧什么"
- 核心价值分权重永远 ≥ 辅助主题分权重 × 2（防跑偏硬约束）
- 不允许主题热度推动核心质量不合格的股票进入前排
- 所有推荐输出必须附带"为什么它值得看"和"为什么现在不一定值得买"的双面说明

**代码约束**：`coreWeight >= 0.55`（不可下调至此线以下）

### 1.2 买股票就是买公司

**哲学含义**：买股票是买公司的一部分，不是买一张会波动的纸。真懂这句话的人可能不到 1%。

**系统落地**：
- 评分的第一维度永远是"生意模式质量"——这家公司做的是什么生意？这个生意好不好？
- 所有技术指标（MACD/BOLL/KDJ）仅作为辅助参考，不参与核心评分
- 系统定位为"辅助决策工具"，输出的是"值得研究的候选"，而非买卖信号

### 1.3 能力圈：知道自己不知道什么

**哲学含义**：知道自己能力圈有多大，远比能力圈有多大重要得多。投资和做事之前，先诚实评估自己是否真的"懂"。

**系统落地**：
- 用户可以标记"我懂的行业"（能力圈范围），系统在能力圈内优先排序
- 系统为每只股票输出"理解难度"标记（商业模式简单/中等/复杂/极复杂）
- 不在能力圈内的公司不会被剔除，但会标记"圈外"并降权展示
- 对于商业模式极复杂或财务异常的公司，系统主动提示"需要深入研究"

**判断标准**："懂"的标准 = 你能判断这家公司 10 年后大概会怎样。如果不能，就承认不懂。

### 1.4 不为清单：知道不做什么比做什么更重要

**哲学含义**：一般人总觉得做什么最重要，其实最重要的是不做什么。巴菲特能有今天最重要的是"不做什么"。

**系统落地 — 三不原则**：
- **不做空**（系统不提供做空/看跌建议）
- **不借钱**（资金适配模块会检查：如果用户需要借钱才能买，降权）
- **不懂不做**（能力圈外的公司标记"圈外"，不作为重点关注推荐）

**系统落地 — 不为清单过滤**：
- 管理层有诚信污点 → 硬过滤
- 生意模式有硬伤 → 硬过滤
- 有息负债率过高（>70%）→ 硬过滤
- 连续 3 年不盈利 → 硬过滤
- ST/退市风险 → 硬过滤

### 1.5 平常心：不被贪婪和恐惧左右

**哲学含义**：在市场恐慌时不恐惧，在市场狂热时不贪婪。平常心就是回到事物本质的心态。

**系统落地**：
- 系统提供"市场情绪温度计"（基于全市场估值分位），但不改变核心评分
- 当市场整体估值处于极端区域时，系统输出额外提示但不变更排序逻辑
- 自动刷新机制（每小时一次）而非实时秒级——避免用户被短期波动牵着走
- 不提供"今日涨幅榜"或"盘中异动"等容易引发情绪波动的功能

## 2. 三角核心评估框架

段永平反复强调：价值投资最重要的三点——对的生意、对的人、对的价格。其中生意模式最重要，对的人其次，对的价格没那么重要。

本系统将这三个维度作为**核心评分的三大支柱**。

### 2.1 生意模式质量（对的生意）— 权重 0.32

**核心理念**：好的生意模式就是能够持续赚取超额利润的模式。它有护城河，竞争对手很难复制。

**评估维度**：

| 子维度 | 权重 | 说明 | 评分来源 |
|--------|------|------|---------|
| 护城河强度 | 0.30 | 品牌/专利/网络效应/转换成本/成本优势 | 行业地位 + 毛利率稳定性 |
| 行业地位 | 0.25 | 行业前3、细分龙头、定价权 | 市值排名 + 毛利率 vs 行业均值 |
| 商业模式稳定性 | 0.25 | 收入可预测性、客户分散度 | 收入波动率 + 大客户依赖度 |
| 产品差异化 | 0.20 | 是否做差异化而非价格战 | 毛利率水平 + 研发投入效率 |

**代理评分公式**（真实数据模式下的轻量计算）：
```
businessModelScore = 
  moatScore(marketCap, grossMargin, industryPosition) × 0.30 +
  positionScore(marketCap, industryRank, pricingPower) × 0.25 +
  stabilityScore(revenueVolatility, customerConcentration) × 0.25 +
  differentiationScore(grossMargin, rdRatio) × 0.20
```

> 未来接入完整财报后，替换为真实护城河评估。

### 2.2 管理与企业文化（对的人）— 权重 0.22

**核心理念**：企业文化不好的公司，生意模式再好也要打折扣。看企业文化就看它说没说过瞎话。

**评估维度**：

| 子维度 | 权重 | 说明 | 评分来源 |
|--------|------|------|---------|
| 诚信记录 | 0.35 | 公开谴责/财务造假/承诺失信 | 监管处罚数据库 |
| 股东回报 | 0.25 | 分红稳定性/回购/股权激励合理性 | 分红历史 + 稀释率 |
| 长期主义 | 0.20 | 研发投入持续性/不追短期热点 | R&D 趋势 + 战略一致性 |
| 管理层稳定 | 0.20 | 核心团队变动/治理结构 | 高管变动频率 |

**诚信一票否决**：若存在财务造假、公开谴责、或严重失信记录，直接标记"不为清单"，不进入推荐池。

**代理评分公式**：
```
managementScore =
  integrityScore(regulatoryFlags, restatementHistory) × 0.35 +
  shareholderScore(dividendHistory, buybackActivity, dilution) × 0.25 +
  longtermismScore(rdTrend, strategicConsistency) × 0.20 +
  stabilityScore(executiveTurnover, governanceStructure) × 0.20
```

### 2.3 估值与安全边际（对的价格）— 权重 0.18

**核心理念**：未来现金流折现是思维方式，不是算法。毛估估就够了。从 10 年后往回看，当前价格是否便宜？

**评估维度**：

| 估值方法 | 说明 | 适用场景 |
|---------|------|---------|
| 盈利收益率法 | 1/PE vs 无风险利率 | 盈利稳定的成熟公司 |
| 股息率法 | 股息率 vs 机会成本（3%） | 高分红公司 |
| 资产折扣法 | PB vs 行业均值 + ROE 调整 | 重资产行业 |
| 毛估估区间 | 保守/中性/乐观三档估值 | 所有公司 |

**毛估估估值区间计算**：
```
intrinsicValue = price × (1 + (valuationScore - 50) / 80)  // 估值中枢
conservativeValue = intrinsicValue × 0.88  // 保守估值（安全边际12%）
optimisticValue = intrinsicValue × 1.12    // 乐观估值

buyZoneLow = conservativeValue × 0.72   // 深度低估买入区
buyZoneHigh = conservativeValue × 0.85  // 合理偏低关注区
```

**五档估值结论**（替代原三档）：

| 结论 | 条件 | 含义 |
|------|------|------|
| 低估 | valuationScore ≥ 78 | 价格显著低于内在价值，安全边际充足 |
| 合理偏低 | 68 ≤ valuationScore < 78 | 价格有一定折让，值得关注 |
| 合理 | 58 ≤ valuationScore < 68 | 价格基本反映价值 |
| 偏高估 | 48 ≤ valuationScore < 58 | 价格略高于价值，需耐心等待 |
| 明显高估 | valuationScore < 48 | 价格严重偏离价值，不建议参与 |

## 3. 财务质量验证

生意模式好、人对、价格合适，还需要财务上站得住。财务质量是"验真"环节。

### 3.1 盈利质量 — 权重 0.10

| 指标 | 理想区间 | 评分逻辑 |
|------|---------|---------|
| ROE | 15%-30% | >20% 满分，<8% 不及格 |
| ROIC | >12% | 反映真实资本回报 |
| 毛利率稳定性 | 波动 <20% | 越稳定越高分 |
| 利润含金量 | 经营现金流/净利润 > 0.8 | "赚到手的才是真的" |

### 3.2 现金流质量 — 权重 0.08

| 指标 | 理想区间 | 评分逻辑 |
|------|---------|---------|
| 经营现金流 | 持续为正 | 连续3年为负则标记风险 |
| 自由现金流 | > 净利润 × 0.6 | FCF 是"真正可以拿走"的钱 |
| 资本支出效率 | CAPEX/经营现金流 < 0.5 | 轻资产模式加分 |

### 3.3 资产负债安全 — 权重 0.06

段永平的"不借钱"原则延伸到投资标的：好公司不需要大量借债。

| 指标 | 警戒线 | 评分逻辑 |
|------|--------|---------|
| 有息负债率 | <40% | >70% 直接标记"不为清单" |
| 利息保障倍数 | >5x | <2x 标记风险 |
| 短债/现金 | <1.0 | >1.5 标记流动性风险 |

### 3.4 经营稳定性 — 权重 0.04

| 指标 | 说明 |
|------|------|
| 近5年收入波动率 | 越低越好 |
| 近5年利润波动率 | 越低越好 |
| 是否有重大会计变更 | 有则标记 |
| 审计意见类型 | 非标意见直接标记风险 |

## 4. 辅助判断维度

这些维度提供**辅助排序**作用，不能推翻核心评分的结论。

### 4.1 行业景气度 — 辅助权重 0.40

当前行业处于上升期/稳定期/下行期。基于行业整体收入增速、政策环境、机构配置比例。

**硬约束**：行业景气度为"下行"时，该行业内公司即使总分高，也不会被列为"重点关注"。

### 4.2 概念热度 — 辅助权重 0.25

市场关注度和主题热度。基于概念关联数量、资金流入、媒体提及频率。

**硬约束**：概念热度评分 > 85 时触发"过热预警"，不会因此提升排序。

### 4.3 催化因素 — 辅助权重 0.35

短期内可能影响基本面的具体事件：业绩拐点、新产品、政策利好、行业整合。

## 5. 资金适配

针对小资金（几千到几万）、高集中（1-2 只）持仓场景。

### 5.1 评估维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 买入门槛友好度 | 0.40 | 一手价格是否在用户资金范围内 |
| 流动性适配 | 0.25 | 换手率/市值是否支持正常买卖 |
| 集中持仓适配 | 0.20 | 波动率和基本面稳定性是否适合重仓 |
| 波动适配 | 0.15 | 日内/周度波动是否在用户承受范围内 |

### 5.2 核心原则

- 如果一手价格超过用户可用资金的 40%，标记"资金不适配"
- 如果一手价格超过用户可用资金，标记"无法买入"
- 高集中持仓场景下，优先推荐波动适中、基本面扎实的公司

## 6. 不为清单与风险过滤

### 6.1 硬性排除（不可关闭）

以下任一条件触发，直接排除出推荐池：

| 过滤条件 | 检测方式 | 哲学来源 |
|---------|---------|---------|
| ST / *ST / 退市风险 | 名称 & 代码检查 | — |
| 上市时间 < 3 年 | listingYears < 3 | — |
| 连续 3 年不盈利 | recentProfitYears ≤ 0 | 不做不懂的生意 |
| 财务数据严重缺失 | hasCompleteFinancials = false | — |
| 管理层诚信污点 | 公开谴责/财务造假/处罚记录 | 不为清单 |
| 有息负债率 > 70% | interestDebtRatio > 0.7 | 不借钱原则 |
| 审计非标意见 | auditOpinion ≠ "标准无保留" | — |

### 6.2 风险标记（参与评分调整但不排除）

| 标记 | 触发条件 | 影响 |
|------|---------|------|
| 估值偏高 | valuationScore < 52 | 不为"重点关注"，标记"估值偏贵" |
| 现金流偏弱 | cashFlowScore < 55 | 风险项 +1 |
| 负债承压 | balanceSheetScore < 55 | 风险项 +1 |
| 经营波动偏大 | stabilityScore < 52 | 风险项 +1 |
| 概念过热 | conceptHeat > 85 | 触发"过热预警" |
| 圈外 | 用户标记不在能力圈内 | 不列入"重点关注" |
| 模式复杂 | businessComplexity = "复杂"或"极复杂" | 标记"需深入研究" |

### 6.3 能力圈边界

用户可在设置中标记"我懂的行业"（能力圈清单）。系统行为：
- 能力圈内公司 → 正常评分排序
- 能力圈外公司 → 附加"圈外"标记，不列入"重点关注"
- 未设置能力圈时 → 全部正常评分，但首次使用提示用户设定

## 7. 多视角共识与置信度

单一评分维度可能偏颇。本系统引入多视角交叉验证机制（取自 Value Analyzer，适配为自动化）。

### 7.1 四维视角

| 视角 | 数据来源 | 方向判断 |
|------|---------|---------|
| 基本面视角 | 核心评分 + 财务质量 | 看多/中性/看空 |
| 估值视角 | 估值评分 + 安全边际 | 低估/合理/高估 |
| 情绪面视角 | 概念热度 + 资金流向 | 冷清/正常/过热 |
| 技术面视角 | MACD/BOLL/KDJ/成交量 | 多头/震荡/空头 |

### 7.2 共识强度

```
四维方向一致（如全看多/全看空）    → 高共识   → 置信度 = high
三维一致、一维中性                  → 中高共识 → 置信度 = medium-high
三维一致、一维相反                  → 低共识   → 置信度 = low（需要深入研究）
两两分歧                           → 极低共识 → 置信度 = very-low（强烈建议谨慎）
```

### 7.3 反向信号

当基本面优秀但情绪面极冷清时 → 可能的机会（"别人恐惧时贪婪"）
当概念极热但基本面薄弱时 → 危险信号（"别人贪婪时恐惧"）

系统在以下场景输出特殊提示：
- 核心评分 ≥ 75 且 conceptHeat ≤ 25 → "市场冷落的好公司"
- 核心评分 < 55 且 conceptHeat ≥ 80 → "热度驱动力大于基本面"

## 8. 总评分公式

### 8.1 默认权重

```
总分 = 
  核心价值分 × 0.62     // 三角核心 + 财务质量
+ 辅助主题分 × 0.18     // 行业景气/概念热度/催化
+ 资金适配分 × 0.12     // 小资金适配
+ 共识调整分 × 0.08     // 多视角一致性（新增）
```

### 8.2 核心价值分权重

```
核心价值分 = 
  businessModelScore  × 0.32   // 对的生意
+ managementScore     × 0.22   // 对的人
+ valuationScore      × 0.18   // 对的价格
+ profitabilityScore  × 0.10   // 盈利质量
+ cashFlowScore       × 0.08   // 现金流质量
+ balanceSheetScore   × 0.06   // 资产负债安全
+ stabilityScore      × 0.04   // 经营稳定性
```

### 8.3 硬约束（不可突破）

1. **核心价值分 < 55 的股票，无论辅助分多高，不得进入 Top 50**
2. **核心价值分权重在任何设置下不得低于 0.55**
3. **生意模式质量权重在任何设置下不得低于 0.22**
4. **触发"不为清单"任一条件的股票，直接排除**
5. **概念热度 > 85 的股票，辅助主题分降权 50%**

## 9. 估值与决策输出

### 9.1 毛估估估值卡

每只股票输出：

| 字段 | 含义 |
|------|------|
| 估值结论 | 五档：低估/合理偏低/合理/偏高估/明显高估 |
| 保守估值 | 保守假设下的每股价值 |
| 中性估值 | 中性假设下的每股价值 |
| 乐观估值 | 乐观假设下的每股价值 |
| 建议关注区间 | [保守估值 × 0.72, 保守估值 × 0.85] |
| 安全边际 | (中性估值 - 当前价格) / 中性估值 |
| 机会成本比较 | 盈利收益率 vs 无风险利率 |

### 9.2 总评分对应的决策语言

| 总分区间 | 结论 | 说明 |
|---------|------|------|
| ≥ 82 | 重点关注 | 生意好、人对、价格合适、财务扎实 |
| 72-81 | 值得观察 | 整体优秀，但某一方面需要进一步确认 |
| 62-71 | 继续研究 | 有亮点但存在需要深入理解的方面 |
| 52-61 | 暂不推荐 | 多个维度存在隐忧 |
| < 52 | 明确回避 | 触发了不为清单或多项硬伤 |

### 9.3 重点关注筛选规则（1-2 只）

在"重点关注"候选池中进一步精选：

1. 核心价值分 ≥ 78
2. 风险标记 ≤ 1 个
3. 估值结论不是"明显高估"或"偏高估"
4. 共识强度 ≥ medium-high
5. 资金适配分 ≥ 60
6. 在用户能力圈内
7. 如果合格标的不足，可以只给 1 只，甚至不给满 2 只

## 10. 系统设计原则

### 10.1 诚实原则

- 不懂就是不懂，不假装懂。商业模式极复杂的公司标记"需深入研究"而非强行评分
- 数据缺失时如实说明，不编造数据
- 每项评分附带置信度，低置信度意味着"这个分数可能是错的"

### 10.2 减法原则

- 宁可错过，不要做错。不为清单优先于"可能的机会"
- 系统功能做减法——不加未经哲学验证的新维度
- 权重调整界面设硬下限，防止用户把系统调成"热点排序器"

### 10.3 长期原则

- 所有判断基于"10 年后会怎样"而非"明天会怎样"
- 不提供日内/盘中/秒级功能
- 刷新频率 1 小时，不是 1 分钟

### 10.4 本分原则

- 系统定位是"辅助决策"，不是"代替决策"
- 所有输出是"值得研究的候选"，不是买卖指令
- 推荐附带风险说明，禁止只说好的不说坏的

## 附录 A：与 Value Analyzer 的关系

Scope 吸收了 Value Analyzer 的以下方法论：
- 多视角分析框架 → 适配为自动化"共识强度"计算
- 硬性规则（RSI>80 不买等） → 融入不为清单和风险标记
- 结构化决策看板 → 体现在估值卡和结论语言

Scope 与 Value Analyzer 的根本区别：
- Value Analyzer 是**单股深度分析工具**（手动触发，一只有一只）
- Scope 是**全市场自动筛选系统**（持续运行，全局排序）

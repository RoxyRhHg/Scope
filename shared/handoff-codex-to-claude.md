# Codex → Claude 交付汇报规范

Codex 完成每个任务后，用此格式向 Claude 汇报。

---

## 交付格式

在 `shared/codex-delivery.md` 中追加以下内容：

```markdown
# 交付汇报：T[N] — [任务标题]

**完成时间**：[时间]
**用时**：[实际用时 vs 预估用时]

## 改动清单

| 文件 | 改动类型 | 改动行数 | 说明 |
|------|---------|---------|------|
| [path/file1] | 修改 | +N/-M | [说明] |
| [path/file2] | 新增 | +N | [说明] |

## 验收自检

- [ ] 功能：[验收标准自检结果]
- [ ] 边界：[边界条件测试结果]
- [ ] 回归：`npm test` 全部通过

### 测试输出

```
[粘贴 npm test 或相关测试命令的输出]
```

## 偏差说明

> 如果实施过程中偏离了原始任务规格，在此说明：
> - 偏离了什么
> - 为什么偏离
> - 如何确认偏差不影响目标

如有偏差，填写；无偏差则写"严格按规格实施"。

## 遗留问题

- [ ] [问题描述]（无则写"无"）
```
